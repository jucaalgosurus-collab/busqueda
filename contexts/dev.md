# HJC Development Context

Active development mode. Write code first, explain after. Get it working, then right, then clean.

## Priorities
1. Ship working code
2. Then make it correct
3. Then make it clean

## Always
- Preserve state in memory/state/active-state.md
- Use sprint contracts for non-trivial work
- Run evaluator after every code generation sprint
- Keep instincts updated

## Never
- Skip the evaluator
- Lose the objective
- Expand scope beyond the sprint contract
- Self-praise mediocre work
